# 03. 실행 로그

## 1. 시나리오 설명

본 실행 로그는 “데이터 분석 결과 기반 보고서 개요 생성” 업무 자동화 프롬프트가 실제 대화 상황에서 안정적으로 작동하는지 검증하기 위해 작성되었다.

AI는 시스템 설계 문서에서 정의한 “리포트 코치” 역할을 수행한다. 리포트 코치는 사용자가 제공한 데이터 분석 결과를 바탕으로 보고서 제목, 한 줄 요약, 핵심 인사이트, 보고서 목차, 리스크 및 한계, 후속 분석 과제, 확인 필요 사항을 구조화하는 업무 보고서 설계 어시스턴트이다.

이번 실행 로그의 업무 시나리오는 2026년 1월~3월 웹 서비스 가입자 및 결제 사용자 데이터를 바탕으로, 최근 3개월간 유료 전환율 하락 원인을 점검하고 경영진 보고용 보고서 개요를 작성하는 것이다.

초기 입력 데이터에는 다음 내용이 포함되었다.

- 전체 유료 전환율: 4.8% → 3.9%
- 신규 가입자 수: 12% 증가
- 무료 체험 후 결제 전환율: 18% 감소
- 모바일 사용자의 이탈률: 7%p 증가
- PC 사용자의 이탈률: 2%p 증가
- 무료 체험 종료 직전 이탈 비중이 가장 높음
- 가격, 경쟁사, 광고비 데이터는 제공되지 않음

대화는 단순히 보고서 개요를 한 번 생성하는 데 그치지 않고, 사용자가 중간에 분석 목적과 보고 대상을 추가하고, Few-shot 예시를 제공하며, 단계적 검토 조건을 추가하는 방식으로 진행되었다. 이를 통해 AI가 사용자의 추가 조건을 반영해 출력 형식을 개선하는지 확인하였다.

또한 대화 후반부에서는 환각 검증 질문을 포함하였다. 사용자는 가격 인상이 유료 전환율 하락의 가장 큰 원인인지, 신규 가입자가 증가했으므로 유료 사용자도 증가했다고 볼 수 있는지, 신규 가입자 증가의 가장 큰 원인이 무료 체험인지, 광고비 효율을 판단할 수 있는지 등을 질문하였다.

이 과정에서 AI는 입력 데이터에 없는 원인이나 수치를 단정하지 않고, “현재 데이터만으로는 단정할 수 없음”, “확인 필요”, “추가 데이터 필요”로 구분하여 답변하였다.

마지막으로 사용자는 최종 시스템 프롬프트를 다시 제공하고, 지금까지의 대화 내용을 반영한 최종 보고서 개요를 요청하였다. AI는 기존 입력 데이터와 중간 검증 내용을 유지하면서 경영진 보고용 최종 보고서 개요를 작성하였다.

따라서 본 실행 로그는 다음 항목을 검증하는 데 목적이 있다.

1. 10턴 이상의 대화에서 초기 데이터와 분석 목적을 유지하는지 확인
2. 중간에 추가된 보고 대상, Few-shot 예시, 작성 조건을 반영하는지 확인
3. 입력 데이터에 없는 가격, 경쟁사, 광고비 관련 원인을 단정하지 않는지 확인
4. 신규 가입자 증가와 유료 사용자 증가를 혼동하지 않는지 확인
5. 모바일과 PC 이탈률 증가 폭 차이를 정확히 계산하는지 확인
6. 최종 출력이 시스템 프롬프트의 보고서 개요 형식을 따르는지 확인

## 2. 문제 지점과 개선 요약

### 2.1 문제 지점 1: 과도하게 장황한 초기 출력

#### 상황

1턴에서 사용자는 2026년 1월~3월 웹 서비스 가입자 및 결제 사용자 데이터를 제공하고 보고서 개요 작성을 요청하였다. AI는 보고서 목적, 분석 범위, 핵심 요약, 주요 지표 변화, 발견 사항, 해석 및 시사점, 추가 확인 데이터, 권장 분석 방향, 결론까지 포함한 상세한 보고서 개요를 생성하였다.

초기 출력은 분석 내용이 풍부했지만, 경영진 보고용으로는 다소 길고 구조가 복잡했다. 또한 사용자가 요구한 분석 목적과 보고 대상이 첫 요청에 명시되지 않아, 출력의 초점이 일반 분석 보고서에 가까웠다.

#### 개선 과정

사용자는 이후 분석 목적을 “최근 3개월간 유료 전환율 하락 원인 파악 및 후속 분석 방향 정리”로 명시하고, 보고 대상을 “경영진”으로 지정하였다.

AI는 이를 반영하여 경영진 의사결정에 필요한 질문, 핵심 지표 변화, 주요 발견 사항, 후속 분석 우선순위 중심으로 보고서 구조를 수정하였다.

#### 개선 결과

초기 일반 보고서 개요가 경영진 보고용 구조로 개선되었다. 특히 유료 전환율 하락 원인 후보, 무료 체험 종료 구간, 모바일 이탈 증가, 외부 요인 판단 한계가 더 명확하게 정리되었다.

### 2.2 문제 지점 2: Few-shot 예시 반영에 따른 출력 형식 개선

#### 상황

사용자는 대화 중간에 좋은 입력과 좋은 출력 예시 3개를 제공하였다. 예시에는 유료 전환율 하락 분석, 캠페인 성과 분석, 모호한 입력 케이스가 포함되었다.

이전 출력은 내용은 충분했지만, 과제에서 요구하는 “보고서 제목 → 한 줄 요약 → 핵심 인사이트 → 목차 → 리스크 및 한계 → 후속 분석 제안 → 확인 필요 사항” 형식과 완전히 일치하지는 않았다.

#### 개선 과정

AI는 사용자가 제공한 Few-shot 예시를 참고하여 보고서 개요를 재작성하였다. 이후 출력은 보고서 제목, 한 줄 요약, 핵심 인사이트, 권장 보고서 목차, 리스크 및 한계, 후속 분석 제안, 확인 필요 사항으로 정리되었다.

#### 개선 결과

Few-shot 예시를 제공한 이후 출력 형식이 더 안정적으로 바뀌었다. 특히 핵심 인사이트마다 근거가 붙었고, 가격·경쟁사·광고비처럼 제공되지 않은 데이터는 리스크 및 확인 필요 사항으로 분리되었다.

## 3. 최종 수정 결과 요약

실행 로그를 통해 최종적으로 다음과 같은 개선이 이루어졌다.

1. 초기 보고서 개요는 상세 분석 중심이었으나, 이후 경영진 보고용으로 구조와 문체가 조정되었다.
2. Few-shot 예시를 제공한 뒤, 출력 형식이 보고서 개요 템플릿에 더 가깝게 개선되었다.

결론적으로 본 실행 로그는 설계한 시스템 프롬프트가 10턴 이상의 대화에서도 초기 맥락을 유지하고, 중간 조건과 추가 검증 질문을 반영하며, 입력 데이터에 없는 내용을 단정하지 않도록 작동한다는 점을 보여준다.
