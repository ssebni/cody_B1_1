# 01. LLM 모델 비교·선정 보고서

## 1. 비교 목적

본 보고서는 동일한 업무 과업을 3종 이상의 LLM에 입력한 뒤, 모델별 결과 품질을 비교하고 최종적으로 가장 적합한 모델을 선정하기 위해 작성되었다.

비교 대상 업무는 “데이터 분석 결과를 바탕으로 경영진 보고용 보고서 개요를 생성하는 작업”이다. 이 과업은 입력된 수치와 관찰 내용을 기반으로 보고서 제목, 한 줄 요약, 핵심 인사이트, 보고서 목차, 리스크, 후속 분석 과제, 확인 필요 사항을 구조화하는 것을 목표로 한다.

이 과업에서는 특히 입력 데이터에 없는 사실을 임의로 만들어내지 않는 것이 중요하다. 따라서 정확성, 형식 준수, 업무 적합성, 한국어 표현, 환각 방지 능력을 중심으로 모델을 평가하였다.

## 3. 동일 입력 프롬프트

아래 프롬프트를 세 모델에 동일하게 입력

\[업무 과업\]

너는 데이터 분석 결과를 바탕으로 업무 보고서 개요를 작성하는 AI 어시스턴트다.

아래 분석 결과를 바탕으로 경영진 보고용 보고서 개요를 작성해줘.

\[분석 목적\]

최근 3개월간 유료 전환율 하락 원인을 파악하고, 후속 분석 방향을 정리한다.

\[보고 대상\]

경영진

\[데이터 범위\]

2026년 1월~3월 웹 서비스 가입자 및 결제 사용자 데이터

\[주요 수치\]

- 전체 유료 전환율: 4.8% → 3.9%

- 신규 가입자 수: 12% 증가

- 무료 체험 후 결제 전환율: 18% 감소

- 모바일 사용자의 이탈률: 7%p 증가

- PC 사용자의 이탈률: 2%p 증가

- 무료 체험 종료 직전 이탈 비중이 가장 높음

\[관찰된 패턴\]

- 신규 가입자는 증가했지만 유료 전환율은 하락했다.

- 모바일 사용자의 이탈률 증가 폭이 PC 사용자보다 크다.

- 무료 체험 종료 직전 이탈이 집중되어 있다.

- 가격, 경쟁사, 광고비 데이터는 제공되지 않았다.

\[원하는 출력 형식\]

다음 항목을 포함해서 작성해줘.

1.  보고서 제목

2.  한 줄 요약

3.  핵심 인사이트 3개

4.  보고서 목차

5.  리스크 및 한계 3개

6.  후속 분석 제안 3개

7.  확인 필요 사항

\[작성 조건\]

- 경영진 보고용으로 간결하고 객관적인 문체를 사용한다.

- 입력 데이터에 없는 원인을 단정하지 않는다.

- 근거 없는 수치를 새로 만들지 않는다.

- 가격, 경쟁사, 광고비와 관련된 결론은 “확인 필요”로 표시한다.

- 각 인사이트에는 입력 데이터에 근거한 설명을 붙인다.

## 4. 평가 기준

| 평가 축 | 평가 내용                                 | 점수 기준 |
|-------------|-----------------------------------------------|---------------|
| 정확성      | 입력 데이터의 수치와 내용을 정확히 반영했는가 | 1~5점         |
| 형식 준수   | 요청한 7개 출력 항목을 모두 포함했는가        | 1~5점         |
| 업무 적합성 | 경영진 보고용 보고서 개요로 활용 가능한가     | 1~5점         |
| 한국어 표현 | 문장이 자연스럽고 업무 문체에 적합한가        | 1~5점         |
| 환각 방지   | 제공되지 않은 정보에 대해 단정하지 않았는가   | 1~5점         |

## 5. 모델별 결과 요약

### 5.1 ChatGPT 결과 요약

ChatGPT는 요청한 출력 형식을 전반적으로 잘 따랐으며, 보고서 제목, 한 줄 요약, 핵심 인사이트, 목차, 리스크, 후속 분석 제안을 구조적으로 제시하였다. 입력 데이터에 없는 가격, 경쟁사, 광고비 관련 내용은 단정하지 않고 확인 필요 항목으로 분리하였다.

다만 일부 표현은 다소 일반적인 업무 보고 문체에 가까웠으며, 인사이트별 근거 설명의 깊이는 추가 개선 여지가 있었다.

### 5.2 Claude 결과 요약

Claude는 문장 표현이 자연스럽고 보고서 문체가 안정적이었다. 특히 경영진이 읽기 쉬운 형태로 핵심 메시지를 정리하는 데 강점이 있었다. 리스크와 한계 항목도 비교적 신중하게 작성되었다.

다만 요청한 출력 항목 중 일부 항목의 구분이 명확하지 않거나, 목차와 후속 분석 제안이 일부 중복되는 경향이 있었다.

### 5.3 Gemini 결과 요약

Gemini는 핵심 수치를 빠르게 반영하고 간결한 요약을 생성하였다. 전체적으로 요청한 과업을 수행했으나, 일부 항목에서 입력 데이터에 없는 원인을 다소 추정하는 표현이 나타났다.

예를 들어 가격이나 경쟁 환경에 대한 데이터가 없는데도 이를 원인 가능성으로 강하게 제시할 경우, 환각 위험이 있다고 판단하였다. 따라서 사실 기반 보고서 작성 과업에서는 추가 검증이 필요하다.

## 7. 최종 선정 모델

최종 선정 모델은 ChatGPT이다.

ChatGPT를 선정한 이유는 다음과 같다.

첫째, 입력 데이터에 포함된 수치와 관찰된 패턴을 비교적 정확하게 반영하였다. 특히 전체 유료 전환율 하락, 신규 가입자 증가, 모바일 이탈률 증가, 무료 체험 종료 직전 이탈 집중이라는 핵심 내용을 빠짐없이 반영하였다.

둘째, 요청한 출력 형식을 가장 안정적으로 준수하였다. 보고서 제목, 한 줄 요약, 핵심 인사이트, 목차, 리스크, 후속 분석, 확인 필요 사항을 명확히 구분하여 제출물 형식으로 활용하기 쉬웠다.

셋째, 환각 방지 측면에서 가장 안정적이었다. 가격, 경쟁사, 광고비 데이터가 제공되지 않았다는 조건을 반영하여, 해당 내용을 단정하지 않고 “확인 필요”로 분리하였다. 이는 사실 기반 업무 보고서 작성 과업에서 중요한 장점이다.

따라서 본 과업인 “데이터 분석 결과 기반 보고서 개요 생성”에는 ChatGPT가 가장 적합한 모델로 판단된다.

## 8. 한계 및 보완점

본 비교는 웹 환경에서 기본 설정으로 수행한 결과를 바탕으로 하였기 때문에, API 환경이나 세부 파라미터 설정에 따라 결과가 달라질 수 있다.

또한 각 모델의 응답은 동일한 프롬프트를 기준으로 비교했지만, 모델별로 최적화된 프롬프트를 따로 설계하면 성능 차이가 달라질 수 있다.

향후에는 더 많은 테스트 케이스를 적용하고, 실제 업무 데이터와 검증 기준을 추가하여 평가 신뢰도를 높일 필요가 있다.
